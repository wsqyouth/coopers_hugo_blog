## Front Matter 字段说明

```yaml
---
title: "文章标题"                    # 必填：文章的标题
date: "2025-01-01"                  # 必填：发布日期，格式 YYYY-MM-DD
author: "Cooper"                    # 必填：作者名称
categories: ["分类名"]               # 必填：文章分类，数组格式
tags: ["标签1", "标签2"]            # 必填：文章标签，数组格式
description: "文章简短描述"         # 必填：SEO描述，100-160字符
slug: "url-slug"                    # 必填：URL路径，建议使用英文
draft: false                        # 必填：是否为草稿，true=不发布
featured: false                     # 必填：是否为精选文章
---
```

## 文章结构建议

### 1. 引言部分

用1-2段文字简要介绍文章主题和要解决的问题。

**示例**：
> 本文将介绍如何使用Hugo搭建个人博客，从环境准备到部署上线的完整流程。

### 2. 主要内容

使用多级标题组织内容：

#### 二级标题（##）

用于主要章节分隔。

#### 三级标题（###）

用于小节内容。

#### 四级标题（####）

用于更细的内容划分（谨慎使用）。

### 3. 代码示例

技术文章中的代码块：

```javascript
// JavaScript 代码示例
function greet(name) {
    return `Hello, ${name}!`;
}

console.log(greet('World'));
```

```python
# Python 代码示例
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

print(fibonacci(10))
```

### 4. 列表和引用

**无序列表**：
- 第一点
- 第二点
- 第三点

**有序列表**：
1. 第一步
2. 第二步
3. 第三步

**引用块**：
> 这是一段引用内容，可以用来突出重点或引用他人观点。

### 5. 表格

| 特性 | 说明 | 优势 |
|------|------|------|
| 静态生成 | 预渲染HTML | 速度快 |
| Markdown | 简单语法 | 易编写 |
| 主题系统 | 可定制 | 灵活 |

### 6. 图片

```markdown

```

### 7. 链接

- 内部链接：[关于页面](/about)
- 外部链接：[Hugo官网](https://gohugo.io)

## 写作建议

### 标题规范

1. 文章标题简洁明了（5-15字）
2. 每个章节标题都有意义
3. 避免过深的层级（最多4级）

### 内容组织

1. **开头**：简要说明背景和目的
2. **正文**：层次分明，逻辑清晰
3. **总结**：回顾要点，提供展望

### SEO优化

1. description字段简洁有力
2. 合理使用标签（3-5个）
3. slug使用英文短语（用-分隔）

### 代码规范

1. 选择正确的语言标识符
2. 添加必要的注释
3. 保持代码简洁可读
